# Practice Exercises

These exercises are meant to make the ideas stick.
Do them slowly. You are not supposed to rush.

## Exercise 1: Describe A Person

Write down a type for a person with:

- a required name
- a required age
- an optional nickname

## Exercise 2: Choose The Right Type

Decide whether each value should be a `string`, `number`, `boolean`, union, or object shape:

- a product name
- a login state
- a cart total
- a yes/no answer

## Exercise 3: Pick A Function Shape

Design a function that:

- takes a list of numbers
- returns the largest one

Think about what should happen if the list is empty.

## Exercise 4: Narrow A Union

Imagine a value that can be `string | number`.
Write out how you would safely handle both cases in plain English.

## Exercise 5: Choose Between `unknown` And `any`

Describe a situation where data comes from outside your program.
Which type would you prefer at the boundary, and why?

## Exercise 6: Model A Simple API Response

Design a type for a response that contains:

- `success`
- `data`
- an optional `errorMessage`

## Exercise 7: Think About Async

Write a one-sentence explanation of what a promise represents.

## Exercise 8: Debugging Practice

Write down a mistake you have made before in any language.
Then answer:

- What assumption was wrong?
- How could TypeScript have helped?
