# 00. Start Here

TypeScript can feel big at first, so this lesson gives you the mental model before the syntax.

## What TypeScript Is

TypeScript is JavaScript plus a type system.
That means you still write JavaScript-style code, but you also give the compiler extra information about what kind of values your code expects.

TypeScript does not replace JavaScript.
It checks your code before it runs.

## Why TypeScript Exists

JavaScript is flexible, but that flexibility can hide mistakes.
For example, if a value should be a number and you accidentally pass a string, JavaScript may not complain until later.

TypeScript helps catch mistakes earlier.
That saves time, especially in larger programs where one small bug can spread.

## The Big Idea

Think of TypeScript as a careful assistant.
It reads your code and asks:

- Is this value the kind I expected?
- Can this property actually exist?
- What happens if this array is empty?
- Are you handling all possible cases?

That is why TypeScript feels helpful instead of annoying once you get used to it.

## What TypeScript Is Not

TypeScript is not a separate programming language you run directly.
The browser or Node.js runs JavaScript.
TypeScript is compiled into JavaScript first.

TypeScript is also not a guarantee that your program is perfect.
It is a strong safety net, not a magic shield.

## How To Read The Course

If a lesson mentions a word you do not know, do not panic.
Read the whole lesson once, then come back and reread the confusing part.
Programming knowledge builds by repetition.

## Quick Recap

- TypeScript is JavaScript with extra type checking.
- It catches many mistakes before the code runs.
- It helps most when programs become bigger and more complex.
- It does not replace JavaScript; it improves the development experience.

## Common Mistakes

- Thinking TypeScript runs instead of JavaScript.
- Trying to memorize all syntax at once.
- Skipping the examples and only reading the words.
