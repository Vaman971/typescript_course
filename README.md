# TypeScript Course

This folder is a beginner-friendly TypeScript course for someone who is new to programming.
It is designed to be read in order, but each lesson also works as a future refresher.

## How To Use This Course

1. Start with `00-start-here.md`.
2. Read the lessons in order.
3. Open the matching `example-*.ts` files when a lesson points to them.
4. Try the exercises in `practice-00-exercises.md`.
5. Revisit `cheatsheet.md` when you want a fast refresher later.

## Course Map

- `00-start-here.md` explains how to think about the course.
- `01-what-is-typescript.md` explains what TypeScript is and why it exists.
- `02-values-and-types.md` covers variables, primitives, inference, and annotations.
- `03-objects-arrays-functions.md` covers objects, arrays, and functions.
- `04-unions-literals-narrowing.md` covers unions, literal types, and narrowing.
- `05-types-and-interfaces.md` covers aliases, interfaces, optional properties, readonly, tuples, and enums.
- `06-generics.md` covers generics and reusable code.
- `07-classes.md` covers classes, access modifiers, abstract classes, inheritance, and composition.
- `08-modules-and-tsconfig.md` covers imports, exports, tsconfig, and compiler options.
- `09-advanced-types.md` covers utility types, mapped types, conditional types, `keyof`, `typeof`, indexed access, and template literal types.
- `10-async.md` covers promises and async functions.
- `11-real-world-patterns.md` covers practical patterns used in real projects.
- `12-debugging-and-common-errors.md` covers common beginner mistakes and how to debug them.

## Project Files

- `package.json` provides a simple course workspace setup.
- `tsconfig.json` enables strict TypeScript checking for the examples.
- `cheatsheet.md` gives a fast summary of the most useful concepts.
- `practice-00-exercises.md` gives practice tasks with short prompts.

## Suggested Reading Order

If you are brand new to programming, do not rush.
Read one lesson, then inspect its example file, then do one exercise.
That slower pace helps the ideas stick.

## Why This Folder Exists

The goal is not just to memorize syntax.
The goal is to understand how TypeScript helps you write safer JavaScript, how to read type errors, and how to choose the right tool for the job.

## Good Learning Habit

When a concept feels confusing, ask these three questions:

- What problem does this feature solve?
- What does the code look like before and after using it?
- What mistake is TypeScript trying to protect me from?
